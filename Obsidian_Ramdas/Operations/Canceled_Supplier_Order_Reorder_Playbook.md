# Canceled Supplier Order → Reorder Playbook (MSK)

## 🎯 Цель
Единый алгоритм действий, если supplier заказ был оплачен/создан, но затем отменён и нужен повторный закуп.

---

## ✅ Когда запускать playbook
Запускаем, если есть одновременно:
- письмо/событие от supplier с `canceled`,
- в GE1/PE12 есть `Supplier Order ID`,
- по eBay заказ ещё актуален (не отменён покупателем окончательно).

---

## 🔎 Проверка факта отмены
1. Поиск в почте по `Supplier Order ID`.
2. Подтверждение по теме письма: `Order <ID> has been canceled`.
3. Зафиксировать статус refund:
   - `refund_pending` (ещё не возвращено),
   - `refunded` (возврат завершён).

---

## 📋 Действия в таблице (GE1/PE12)
Обязательные поля:
- `Статус закупа` → `Отменен у поставщика`
- `Статус отправки` → `Без трека`
- Комментарий (в заметке/логе): причина отмены + timestamp.

Новая строка на повторный закуп:
- Создаётся как новый procurement-кейс (снизу PE12, затем авто-подъём наверх).
- Привязка к исходному заказу: `reorder_of=<старый Supplier Order ID>`.

---

## 🧩 Reorder чеклист
1. Подтвердить, что товар/вариант тот же (SKU/qty/item id).
2. Проверить актуальный supplier URL (active/alt).
3. Применить купон из листа `Купоны`.
4. Дойти до pre-pay и отправить скрин Шефу.
5. Оплата только после явного ОК.

---

## ⚠️ Риски
- Двойная закупка (если не проставлен `reorder_of`).
- Повторный заказ на неверный вариант.
- Потеря контроля refund статуса по отменённому supplier order.
